## Priority 0 — Bug Fixes / Foundations

These are correctness issues or foundational gaps that should ship before new feature work.

### 🔵 Bug: Cannot Add or Change Roles After Login (F-18)
**Users:** All | **Effort:** XS
`ProfilePage` renders the memberships list as read-only. There is no way to add a second role, remove a role, or change the primary role type after signup. The `RoleCardPicker` component already handles all of this UI (primary card grid + secondary pill row + remove) but is only used in `SignupPage`. The `updateProfile` store method already accepts a `memberships` patch.

Fix: add an editable "My Roles" section to `ProfilePage` that mounts `RoleCardPicker` initialised from the user's current memberships; on save, call `updateProfile({ memberships, role: primaryRole })`. Admin role must remain non-self-assignable (same constraint as signup).

---

### 🔵 Bug: League Manager Cannot Create a League (F-17)
**Users:** League Manager | **Effort:** XS
A newly registered league manager has no way to create a league. Either the "Create League" action is missing from their UI, the route is not accessible to that role, or the Firestore write rule blocks it. Needs investigation to confirm root cause, then fix: league managers must be able to create and own at least one league on signup/onboarding.

### 🔵 Coach Assignment — Phase 1: Auto-assign + Migration (F-16)
**Users:** Coach, Admin, League Manager | **Effort:** XS
**Full spec:** `.claude/coach-assignment-design-2026-03-26.md`

When a coach creates a team, `coachId` is not auto-populated from `createdBy`. The assign-coach dropdown also performs a full `users` collection scan (won't scale). Three issues to fix in sequence:

**Phase 1 (this sprint):** Auto-set `coachId = createdBy` on team creation when creator's role is `coach`. Run `migrationCoachIdBackfill` to backfill existing teams. No UI changes.

**Phase 2 (next sprint):** Replace the `<Select>` dropdown with a typeahead combobox scoped to the relevant league. Add invite fallback: when a typed email matches no registered coach, offer "Invite [email] as coach" — stores `pendingCoachEmail` on team, sends invitation via existing `sendInvite`. Role-based access: league managers search within their league; admins search platform-wide.

**Phase 3 (after Phase 2):** Safe coach reassignment via `reassignCoach` Cloud Function — atomic update, notifications to both coaches, 30-day read-only access for outgoing coach.

**Phase 4 (parked/premium):** Bulk CSV import of team+coach mappings.

Key constraints: Firestore composite index required `(role, leagueId, displayName)` before Phase 2 ships. Migration script follows same pattern as `migrateUserMemberships`.

## Priority 1 — High Value, High Attraction

These are the most compelling features for acquiring and retaining users. Ship before marketing push.

### 🔵 Player Availability Window (BA Feature 3)
**Users:** Coach, Player | **Effort:** Medium
Players mark date ranges when unavailable; coach sees conflicts when scheduling. Heatmap of availability per player. This directly reduces the #1 coach pain point (attendance uncertainty) and gives players agency.

### 🔵 Post-Game Summary & Result Broadcast (BA Feature 4)
**Users:** Coach, Player, Parent | **Effort:** Small
After recording a result, one-tap broadcast with optional post-game message + Man of the Match callout. Eliminates the group-chat duplication coaches currently do. High-frequency touchpoint — every game is a moment.

### 🔵 "This Week in Sport" Digest (BA Retention Hook 1)
**Users:** All | **Effort:** Small (Cloud Function + email)
Weekly push/email (Monday) summarising the week ahead per user. Coaches see non-responders. Creates a weekly re-engagement loop without requiring app-open. Highest ROI retention mechanic.

### 🔵 Weather Alerts for Outdoor Events (BA Feature 9)
**Users:** Coach, Parent, Player | **Effort:** Small
Open-Meteo API integration. Alert coach 24h before outdoor event if rain probability > 70%. One-tap cancel/confirm. Parents want this badly — it's a concrete, time-saving feature that demonstrates app value on its own.

### 🔵 Absence & Injury Tracker (BA Feature 12)
**Users:** Coach | **Effort:** Small
Coach marks players as injured/suspended with expected return date. Players flagged on roster + event planning views. Private notes. Already partially aligned with Player Availability Window — these can share UI patterns.

## Priority 2 — High Value, Moderate Effort

Important for engagement and differentiation; ship in the sprint after Priority 1.

### 🔵 Milestone Moments (BA Retention Hook 2)
**Users:** All | **Effort:** Small (computed from existing data)
Automated recognition: "Your team just played their 20th game", "Jamie hasn't missed a session in 8 weeks". Home screen banner, dismissible. Especially powerful for parents. No user action required — pure delight.

### 🔵 Streak Indicators for Players (BA Retention Hook 3)
**Users:** Player, Parent | **Effort:** Small
Consecutive events attended shown on player profile. Visible to player, parent, optionally coach. Classic engagement mechanic, creates reason to open app and reason to show up.

### 🔵 Substitute / Guest Player Request (BA Feature 11)
**Users:** Coach | **Effort:** Small
One-off event invite to a non-roster user. Guest RSVP-capable; clearly marked "guest" in attendance; no team data access. Archived after event; optional promotion to full roster member. Solves a real pain point for short-handed teams.

### 🔵 Unified Parent Dashboard / Multi-Child View (BA Feature 2)
**Users:** Parent | **Effort:** Medium
Single dashboard aggregating all linked children's events in chronological order. Colour-coded by child. Inline RSVP. Conflict highlighting for overlapping events. The strongest reason for parents to use the app vs a group chat.

### 🔵 Event Check-In via QR Code (BA Feature 7)
**Users:** Coach, Player | **Effort:** Small
Coach generates time-limited QR code; players scan to self-check-in (deep-link, no app required for scan). Live check-in count for coach. Default 30-minute expiry. Reduces game-day admin friction significantly.

## Priority 3 — Differentiation / Premium Candidates

Features that strengthen retention or unlock monetisation. Plan for next major release.

### 🔵 In-Season Player & Team Stats — *Premium tier*
**Users:** Coach, Player, Parent | **Effort:** Large
Per-game stat entry by coach (sport-configurable stat types: goals, assists, saves, points, etc.). Season aggregate totals per player and team. Stat leaderboard within team. Shareable per-player stat cards (PNG/link). Free tier: attendance stats only. Premium: full per-game stats, leaderboard, export, shareable cards.
> Note: distinct from Season Summary (end-of-season). This is live in-season tracking — coaches enter stats after each event, players see their rolling totals.

### 🔵 Season Summary & Statistics (BA Feature 6) — *Premium candidate*
**Users:** Coach, Player, Parent | **Effort:** Medium
Auto-generated at season close: games/wins/losses, goals, per-player attendance rate. Shareable social card (PNG/link). Free tier: basic totals. Paid tier: per-player breakdown + export. Natural seasonal upsell moment. Pairs with In-Season Stats above.

### 🔵 Post-Season "Year in Review" (BA Retention Hook 5)
**Users:** All | **Effort:** Medium
Spotify Wrapped-style scrollable in-app story per user at season end. Most attended event, biggest win, most active player, etc. Once-a-year word-of-mouth generator. Pairs with Season Summary.

### 🔵 Carpool Coordination (BA Feature 5)
**Users:** Parent, Coach | **Effort:** Medium
Parents offer or request seats for specific events. Match summary visible to all. In-app messaging to confirm arrangements. No address sharing — meeting point is free-text. Solves a high-frequency pain point for youth sport parents.

### 🔵 Drill & Session Planner (BA Feature 8) — *Premium candidate*
**Users:** Coach | **Effort:** Large
Structured session plans attached to practice events (warm-up/drills/cool-down). Each drill: title, duration, description, optional diagram. Personal drill library. Total duration auto-calculated. Free: 10 saved drills. Paid: unlimited + image upload + assistant coach sharing. Highest lock-in of any feature.

### 🔵 GameChanger Integration
**Users:** Coach, League Manager | **Effort:** Medium
Import teams, rosters, and schedules from GameChanger (popular youth baseball/softball platform, owned by Dick's Sporting Goods). Coaches can migrate existing data without re-entering rosters. Read-only pull via GameChanger's unofficial or partner API — no two-way sync in v1. Target sport: baseball/softball primarily, but architecture should be generic enough to extend.
> Note: GameChanger's API is not publicly documented — integration feasibility depends on reverse-engineering or a formal partnership. Flag before scoping.

### 🔵 Team Photo & Media Gallery (BA Feature 10) — *Premium candidate*
**Users:** Parent, Player, Coach | **Effort:** Medium
Per-event photo uploads by any member. Team-level gallery view. Coach pins cover photo. 100 MB free / higher cap on paid tier. Strong parental emotional attachment → upgrade driver.
